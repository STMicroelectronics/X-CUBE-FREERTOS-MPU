X-CUBE-FREERTOS-MPU FreeRTOS Software applications list

The X-CUBE-FREERTOS-MPU package comes with a rich set of applications running on STMicroelectronics boards,

organized by board and provided with preconfigured projects for the main supported toolchains

Projects are structured as follows:

Notes

  1. The FreeRTOS heap size configTOTAL_HEAP_SIZE defined in FreeRTOSConfig.h is set accordingly to the OS resources memory requirements of the application with +10% margin and rounded to the upper Kbyte boundary. For more details about FreeRTOS implementation on STM32Cube, please refer to UM1722 “Developing Applications on STM32Cube with RTOS”.
  2. When used with the MPU, FreeRTOS requires various regions to be defined in the linker file. For more information, refer to the linker files in the example as well as the FreeRTOS Documentation/Blog:
      https://freertos.org/2020/04/using-freertos-on-armv8-m-microcontrollers.html#FREERTOS_WITH_MPU
    • MPU regions are required to have a size which is a multiple of 32 bytes and to be aligned on the 32 byte boundary. In particular, unprivileged task stacks must comply with these requirements.
    • FreeRTOS uses the Systick as time base, thus it is mandatory that the HAL uses a separate time base through the TIM IPs.
